# IoT SIP-based Video Transportation System

## 项目概述

本项目是一个基于SIP（Session Initiation Protocol）协议的物联网视频传输系统，实现了安全的视频通信功能。系统支持IPC（网络摄像机）和NVR（网络视频录像机）设备之间的安全视频传输，集成了国密算法和硬件加密模块，提供端到端的安全通信保障。

## 系统架构

### 核心模块

1. **SIP通信模块** (`sip_com.c/h`)
   - 提供SIP协议基础功能
   - 配置文件读取和网络信息获取
   - 数据编码解码功能

2. **用户代理客户端** (`uac.c/h`)
   - 实现SIP客户端功能
   - 支持注册、邀请、消息发送等操作
   - 密钥协商和令牌交换

3. **用户代理服务器** (`uas.c/h`)
   - 实现SIP服务器功能
   - 处理INVITE、MESSAGE、BYE等请求
   - 事件处理和消息解析

4. **调度模块** (`dispatch.c/h`)
   - 协调UAC和UAS功能
   - 媒体会话管理
   - SDP（Session Description Protocol）处理

5. **接口模块** (`interface.c/h`)
   - 安全协议接口
   - 证书管理和密钥交换
   - 加密算法封装

6. **安全模块** (`secfiles/` 目录)
   - 国密算法实现（SM1/SM2/SM3/SM4）
   - 同方TF09 USB加密模块驱动
   - 硬件加密支持

### 设备类型

- **IPC**: 网络摄像机，视频源设备
- **NVR**: 网络视频录像机，视频接收和存储设备
- **Client**: 客户端设备
- **SIPserver**: SIP服务器

## 功能特性

### 核心功能

1. **SIP注册与认证**
   - 向SIP服务器注册设备
   - 双向身份认证
   - 支持WAPI安全协议

2. **密钥管理**
   - ECDH密钥协商
   - 密钥分发和更新
   - 主密钥派生

3. **令牌交换**
   - 认证令牌（Auth）
   - 重认证令牌（Reauth）
   - 会话结束令牌（Byesession）
   - 连接结束令牌（Byelink）

4. **视频传输**
   - 基于RTP/RTSP的视频流传输
   - SDP协商媒体参数
   - 实时视频播放和录制

5. **安全通信**
   - 端到端加密
   - 数字签名和验证
   - 消息完整性保护

### 安全特性

- 国密算法支持（SM1/SM2/SM3/SM4）
- 硬件加密模块集成
- 证书双向认证
- 前向安全性保障

## 编译与运行

### 依赖库

- **eXosip2**: SIP协议栈库
- **osip2**: OSIP协议库
- **OpenSSL**: 加密算法库
- **pthread**: 线程支持库

### 编译步骤

1. 确保依赖库已安装：
```bash
# Ubuntu/Debian
sudo apt-get install libexosip2-dev libosip2-dev libssl-dev

# CentOS/RHEL
sudo yum install libeXosip2-devel libosip2-devel openssl-devel
```

2. 编译项目：
```bash
cd Debug
make
```

3. 运行程序：
```bash
# 使用默认配置
./sip-system

# 指定配置文件
./sip-system uac.cfg
```

## 配置说明

### 配置文件格式

系统支持三种配置文件：

1. **default.cfg** - 默认配置
2. **uac.cfg** - 用户代理客户端配置
3. **uas.cfg** - 用户代理服务器配置

### 配置参数

```ini
# 服务器配置
server_id=34020000001180000002
server_ip=192.168.17.127
server_port=5060

# 客户端配置
client_id=11111
client_pwd=123456
client_ip=192.168.171.127
client_port=5061

# 设备类型
user_type=CLIENT  # 可选: CLIENT, IPC, NVR

# 默认邀请地址
default_invite=34020000001310000054@192.168.17.127:5062
```

### 证书配置

系统使用X.509证书进行身份认证：
- 证书文件存放在 `cert/` 目录
- 私钥文件存放在 `private/` 目录
- 证书格式为PEM格式

## 使用指南

### 启动程序

程序启动后显示功能菜单：

```
1: register
2: key_nego
3: key distribute
4: token exchange
5: transmit the video between IPC and NVR
6: bye

b: run as a uas
c: exit
```

### 基本操作流程

1. **注册设备** (输入 '1')
   - 向SIP服务器注册设备
   - 完成双向身份认证

2. **密钥协商** (输入 '2')
   - 使用ECDH算法协商会话密钥
   - 生成主密钥和派生密钥

3. **密钥分发** (输入 '3')
   - 向对等设备分发密钥
   - 建立安全连接

4. **令牌交换** (输入 '4')
   - 交换认证令牌
   - 建立安全会话

5. **视频传输** (输入 '5')
   - 启动视频流传输
   - 支持实时播放和录制

6. **结束会话** (输入 '6')
   - 发送BYE消息
   - 关闭媒体会话

### 运行模式

- **UAC模式**: 默认模式，作为客户端运行
- **UAS模式**: 输入 'b' 切换为服务器模式

## 安全机制

### 加密算法

1. **SM1** - 分组密码算法
   - 128位密钥长度
   - 用于数据加密

2. **SM2** - 椭圆曲线公钥密码算法
   - 用于数字签名和密钥交换
   - 支持256位密钥

3. **SM3** - 杂凑算法
   - 256位摘要长度
   - 用于消息完整性验证

4. **SM4** - 分组密码算法
   - 128位密钥长度
   - 用于无线局域网安全

### 密钥管理

- **主密钥（MK）**: 通过ECDH协商生成
- **加密密钥（CK）**: 用于数据加密
- **完整性密钥（IK）**: 用于消息完整性保护
- **密钥加密密钥（KEK）**: 用于密钥保护

### 认证流程

1. 设备向SIP服务器注册
2. 服务器返回挑战（Challenge）
3. 设备使用私钥签名挑战
4. 服务器验证签名并返回认证结果
5. 建立安全会话

## 故障排除

### 常见问题

1. **注册失败**
   - 检查网络连接
   - 验证服务器配置
   - 检查证书和私钥文件

2. **密钥协商失败**
   - 检查ECDH参数
   - 验证对等设备证书
   - 检查随机数生成

3. **视频传输中断**
   - 检查网络带宽
   - 验证RTP/RTSP端口
   - 检查媒体编码格式

4. **加密模块错误**
   - 检查TF09 USB设备连接
   - 验证驱动程序安装
   - 检查权限设置

### 日志分析

- SIP消息日志在控制台输出
- 加密操作日志在安全模块中记录
- 网络错误信息通过系统日志查看
